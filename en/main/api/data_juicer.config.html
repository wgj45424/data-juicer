data_juicer.config.resolve_job_directories(cfg)[source]#

Centralize directory resolution and placeholder substitution. Assumes job_id is already set.

Job Directory Rules: - If work_dir contains ‚Äò{job_id}‚Äô placeholder, it MUST be the last part of the path - Examples:

‚úÖ work_dir: ‚Äú./outputs/my_project/{job_id}‚Äù # Valid ‚úÖ work_dir: ‚Äú/data/experiments/{job_id}‚Äù # Valid ‚ùå work_dir: ‚Äú./outputs/{job_id}/results‚Äù # Invalid - {job_id} not at end ‚ùå work_dir: ‚Äú./{job_id}/outputs/data‚Äù # Invalid - {job_id} not at end

  • If work_dir does NOT contain ‚Äò{job_id}‚Äô, job_id will be appended automatically

  • Examples: work_dir: ‚Äú./outputs/my_project‚Äù ‚Üí work_dir: ‚Äú./outputs/my_project/20250804_143022_abc123‚Äù

After resolution, work_dir will always include job_id at the end.
